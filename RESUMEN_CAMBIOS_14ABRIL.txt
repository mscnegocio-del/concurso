2️⃣ BUG FIX - Historial no se actualizaba al republicar
────────────────────────────────────────────────────────────────────────────────

❌ PROBLEMA:
  • Al publicar una categoría y luego hacerlo de nuevo
  • El historial mostraba SOLO la primera publicación
  • La fecha y usuario NUNCA se actualizaban

🔍 CAUSA RAÍZ:
  • Función SQL: coordinador_avanzar_revelacion_categoria
  • El UPDATE solo ejecutaba si paso_revelacion < puestos_a_premiar
  • Si ya estaba completamente revelada: NO había UPDATE

✅ SOLUCIÓN:
  • Nueva migración: 20260414_001_fix_republish_historial.sql
  • Cambió lógica para SIEMPRE actualizar publicado_at y publicado_por
  • Ahora el historial refleja la ÚLTIMA publicación correctamente

📝 Archivo: supabase/migrations/20260414_001_fix_republish_historial.sql
   → Debe aplicarse en Supabase antes de usar la función

3️⃣ BUG FIX - Pantalla pública no actualiza en tiempo real
────────────────────────────────────────────────────────────────────────────────

❌ PROBLEMA:
  • TV/Proyector (pantalla pública) solo actualiza cada 5 segundos (polling)
  • Latencia observable al publicar una categoría
  • Usuario espera hasta 5 segundos para verlo en pantalla

🔍 CAUSA RAÍZ:
  • PublicoEventoPage NO tenía Realtime subscriptions
  • Solo confiaba en polling cada 5 segundos
  • No había listener para cambios en resultados_publicados

✅ SOLUCIÓN:
  • Agregado Realtime listener en PublicoEventoPage
  • Listener se dispara cuando hay cambios en resultados_publicados
  • Pantalla se actualiza INMEDIATAMENTE (sin esperar 5 segundos)
  • Polling sigue funcionando como fallback

📝 Archivo: src/pages/publico/PublicoEventoPage.tsx
   → useEffect con listener Realtime (líneas 241-253)
   
FLUJO AHORA:
  Admin publica → RPC ejecuta → Realtime notifica → Pantalla actualiza (inmediata)
                                                  ↓ fallback si falla
                                           Polling cada 5 segundos
